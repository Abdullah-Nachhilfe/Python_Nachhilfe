Web-Server

Pfad vs. Query

  • Pfad:
    • einzelne String-Werte
    • Reihenfolge spielt eine Rolle
  • Query:
    • Key/Value Paare, wie ein Dictionary
    • Reihenfolge ist egal

Es gibt Situationen, wo man Formular-Daten NICHT! an die URL zu hängen, z.B.:

  • Daten wie ein Passwort

Es gibt mehrer Methon zur Kommunikation mit dem Server über das HTTP-Protokol. Bis jetzt haben wir die Get-Methode verwendet.

Eine andere Methode ist die Post-Methode

Post vs. Get

  • Get-Request:
    • Ziel: einfacher Anruf einer Resource
    • Wenn wir einen Browser öffnen und etwas eingeben, erfolgt eine Get-Request (Abfrage).
    • Formular-Daten sind Teil der URL
  • POST-Request (Abfrage):
    • Ziel: Senden an Daten an den Server (Geht auch mit Get-Methode) aber hier ist es das Main-Goal
    • Ist nicht dierekt im Browser (über URL) veränderbar. Wie wir es manchmal mit Query gemacht haben.
    • Mit POST-Requests sind die Formular-Daten NICHT! Teil der URL
